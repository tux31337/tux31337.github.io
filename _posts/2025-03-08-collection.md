## **Java 배열 정리: 개념, 장단점, 검색 연산, 객체 배열까지!**

배열(Array)은 **같은 타입의 데이터를 연속된 메모리 공간에 저장하는 자료구조**입니다.  
Java에서 배열은 **고정된 크기**를 가지며, 빠른 데이터 접근이 가능하다는 특징이 있습니다.  
이번 글에서는 **배열의 특징, 장단점, 검색 연산, 데이터 접근 방식, 객체 배열의 차이점**까지 정리해보겠습니다.

---

## **1. 배열이란?**

배열은 **같은 타입의 데이터를 하나의 변수로 관리할 수 있는 자료구조**입니다.  
Java에서 배열을 선언하는 기본적인 방법은 다음과 같습니다.

```java
// 배열 선언 및 초기화
int[] numbers = new int[5];  // 크기가 5인 정수 배열 생성
int[] values = {10, 20, 30, 40, 50};  // 배열 리터럴 초기화
```

배열은 **인덱스(0부터 시작)**를 통해 데이터를 관리합니다.

```java
System.out.println(values[2]);  // 30 출력 (0부터 시작)
```

---

## **2. 배열의 장단점**

### ✅ **배열의 장점**

✔ **빠른 데이터 접근 (O(1))**  
✔ **연속된 메모리 공간을 사용하여 효율적인 캐싱 가능**  
✔ **인덱스를 사용하여 직관적으로 데이터 관리 가능**

### ❌ **배열의 단점**

✘ **크기가 고정됨 (동적 크기 조절 불가)**  
✘ **삽입/삭제 시 비효율적 (O(N))**  
✘ **객체 배열의 경우 추가적인 메모리 접근 필요**

---

## **3. 배열이 빠른 이유 (데이터 접근 속도)**

배열은 **연속된 메모리 공간**을 사용하기 때문에 **주소 계산을 통해 즉시 접근(O(1))**할 수 있습니다.

### **📌 배열의 메모리 주소 공식**

```java
주소 = 배열의 시작 주소 + (인덱스 × 요소 크기)
```

예를 들어, `int` 배열(`int`는 4바이트)의 경우:

```java
int[] arr = {10, 20, 30, 40, 50};
// arr[3]의 메모리 주소 계산:
// 시작 주소가 1000번지라면 → 1000 + (3 × 4) = 1012번지
```

💡 **연속된 메모리 덕분에 특정 요소를 찾는 속도가 매우 빠름! (O(1))**

### **📌 배열 vs 연결 리스트(LinkedList)**

| 비교 항목       | 배열 (Array)     | 연결 리스트 (LinkedList) |
| --------------- | ---------------- | ------------------------ |
| **메모리 구조** | 연속된 공간      | 비연속적 (노드 연결)     |
| **접근 속도**   | O(1) (즉시 접근) | O(N) (포인터 이동 필요)  |
| **삽입/삭제**   | 느림 (O(N))      | 빠름 (O(1))              |

## **4. 배열의 검색 연산 (검색 속도 비교)**

### **📌 1. 선형 검색 (Linear Search) - O(N)**

배열을 처음부터 끝까지 순차적으로 확인하는 방법입니다.  
배열이 **정렬되어 있지 않은 경우**에도 사용할 수 있습니다.

```java
public static int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;  // 찾은 경우 인덱스 반환
        }
    }
    return -1;  // 찾지 못한 경우 -1 반환
}
```

**⏳ 시간 복잡도: O(N)** (최악의 경우 배열 전체를 확인해야 함)

---

### **📌 2. 이진 검색 (Binary Search) - O(log N)**

배열이 **오름차순으로 정렬된 경우** 더 빠르게 검색할 수 있습니다.  
이진 검색은 **중간값을 기준으로 절반씩 범위를 줄여가며 탐색**하는 방식입니다.

```java
import java.util.Arrays;

public static int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (arr[mid] == target) {
            return mid;  // 찾음
        } else if (arr[mid] < target) {
            left = mid + 1;  // 오른쪽 탐색
        } else {
            right = mid - 1;  // 왼쪽 탐색
        }
    }
    return -1;  // 찾지 못한 경우
}
```

**⏳ 시간 복잡도: O(log N)** (탐색 범위가 절반씩 줄어듦)

✔ **Java에서는 `Arrays.binarySearch()` 메서드를 제공함!**

```java
int[] sortedArray = {10, 20, 30, 40, 50};
int index = Arrays.binarySearch(sortedArray, 30);  // index = 2
```

## **5. 객체 배열 (Object Array)의 차이점**

```java
class Person {
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

Person[] people = new Person[3];  // 객체 배열 선언
people[0] = new Person("Alice", 25);
people[1] = new Person("Bob", 30);
people[2] = new Person("Charlie", 35);
```

📌 **객체 배열은 참조값을 통해 추가적인 메모리 접근이 필요하므로 기본 타입 배열보다 상대적으로 느릴 수 있음!**

---

## **6. 배열 정리 요약**

✅ **배열은 연속된 메모리 공간을 사용하여 빠른 데이터 접근(O(1))이 가능**  
✅ **하지만 크기가 고정되어 있고, 삽입/삭제가 비효율적(O(N))**  
✅ **배열 검색 연산은 선형 검색(O(N))과 이진 검색(O(log N)) 방식이 있음**  
✅ **객체 배열은 참조값을 저장하므로 추가적인 메모리 접근이 필요**  
✅ **배열이 아닌 동적 크기 조절이 필요하면 `ArrayList` 같은 컬렉션 사용 고려**
